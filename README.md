# livresgourmands.net

## Objectif de l’étape 01

L’objectif de cette première étape est de :

- Comprendre le cahier des charges
- Identifier les acteurs du système
- Concevoir le diagramme de cas d’utilisation
- Concevoir une première ébauche du diagramme de classes
- Mettre en place la structure du projet sur GitHub

---

## Diagramme de cas d’utilisation

Le diagramme de cas d’utilisation présente les acteurs suivants :

Front Office :
- Internaute
- Client
- Ami

Back Office :
- Éditeur
- Gestionnaire
- Administrateur

Acteur externe :
- Système de paiement sécurisé

Il illustre les principales interactions :
- Recherche et consultation d’ouvrages
- Gestion du panier
- Passage de commande
- Paiement en ligne sécurisé
- Gestion du catalogue et validation des commentaires

---

## Diagramme de classes

Le diagramme de classes comprend les principales entités suivantes :

- Utilisateur (classe abstraite)
- Client
- Administrateur
- Gestionnaire
- Éditeur
- Ouvrage
- Catégorie
- Panier
- Commande
- ListeCadeaux
- Avis
- Commentaire
- Interface Paiement (CarteBancaire, PayPal)

Le modèle respecte les principes de la Programmation Orientée Objet :
- Héritage
- Interface
- Associations avec cardinalités
